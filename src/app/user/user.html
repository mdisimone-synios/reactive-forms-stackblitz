The username validation uses an asynchonous validation. This can be used to call backend API's to check the validity of an input. In this case we have a debounced Observable mimiking this behaviour.

The password validation uses three different validation types. A regular Validators.required, a composed validation that runs multiple validations at once checking the strengs of the password and a combined validation comparing both password formControl`s to make sure they're equal.

A composed validation is a collection of regular validations like minLength or other custom validations. In this example we have wrapped the regular pattern validation into a custom validation in order to return different error codes to show different error messages for each type of pattern (i.e. uppercase, lowercase, specialCharacter...)

The combined validation is different from the others, because it is not bound to the formControl but to the parent formGroup in order to have access to both input field values.
